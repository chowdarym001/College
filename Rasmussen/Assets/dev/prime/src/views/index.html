Overview

This UI toolkit is a highly-modular design system for the rapid development of the Rasmussen College rebranded website. It contains different materials that can be assembled into more complex page layouts.

This guide contains real working examples, code snippets, documentation, and style guidelines.

Development Usage

Start by adding the following files to your page:

<link rel="stylesheet" href="css/styles.css">
<script src="scripts/display.js"></script>

Taxonomies

  1. Atoms:
    • Basic HTML tags that can't be broken down any further without ceasing to be functional
    • Includes abstract elements like color palettes, font stacks, invisible things like animation
    • Eg. form labels, inputs, buttons, links
  2. Molecules:
    • Relatively simple groups of UI elements functioning together as a unit
    • Atoms assembled/combined in specific combination/formation to serve a purpose
    • Eg. form label, search input, and button combined to create a search form
  3. Organisms:
    • Relatively complex UI components composed of groups of molecules and/or atoms, and/or other organisms
    • eg. search form (from molecule example) placed within context of a global header
    • Consist of similar or different molecule types
      • Similar types: categorical page of products, repeated molecules (clothing or programs)
      • Different types: header with logo image, primary navigation, search form
  4. Templates:
    • Page-level objects that place components into a layout and articulate the design's underlying content structure
    • eg. header with search form on homepage
    • Provides context for relatively abstract molecules and organisms
    • Demonstrate how components look/function together in context of layout to prove the parts add up to a well-functioning whole
    • Focus on the page's underlying content structure {rather than the page's final content}
      • Articulate important properties like image sizes and character lengths for headings/text passages
  5. Pages:
    • Specific instances of templates that replace placeholder content with real representative content
    • Essential for testing the effectiveness of the underlying design system
      • Able to take a look at how patterns hold up when real content is applied to the design system
    • Provide a place to articulate variations in templates
